# wikify-work

The work layer is the agent's mutable on-disk scratchpad. It lives
under `<bundle>/work/`. Concept folders carry a ControlCard
(`work.md`), an evidence ledger (`evidence.jsonl`), and a per-concept
advisory claim. Cross-talk between agents flows through the
inbox channels; `work tend` consolidates them deterministically.

## Commands

```
wikify work list                        [--run <b>] [--format text|json]
wikify work list claims                 [--run <b>] [--format text|json]
wikify work list inbox                  [--run <b>] [--format text|json]
wikify work list evidence <concept>     [--run <b>] [--format text|json]
wikify work show     <concept>          [--run <b>] [--full] [--format text|json]
wikify work add concept "<title>"       [--run <b>] [--kind article|person]
                                        [--aliases <json>] [--format text|json]
wikify work add evidence <concept>      --records <jsonl-path> [--run <b>] [--format text|json]
wikify work add feedback <kind>         --record <json|jsonl-path> [--run <b>] [--format text|json]
wikify work set      <concept>          [--status <s>] [--needs-refine] [--run <b>] [--format text|json]
wikify work claim    <concept>          [--owner <id>] [--ttl-seconds N] [--run <b>] [--format text|json]
wikify work release  <concept>          [--owner <id>] [--run <b>] [--format text|json]
wikify work tend                        [--run <b>] [--format text|json]
```

Feedback kinds: `evidence` | `concept` | `merge` | `query`.

## Inputs

- `--run <b>` — bundle override; default is CWD.
- `--records <jsonl-path>` — JSONL of `EvidenceRecord` per line for
  `add evidence`.
- `--record <json|jsonl-path>` — JSON or JSONL for `add feedback`.

## Outputs

- `work/concepts/<slug>/work.md` — ControlCard YAML frontmatter +
  freeform body.
- `work/concepts/<slug>/evidence.jsonl` — append-only evidence ledger.
- `work/concepts/<slug>/.claim` — atomic per-concept claim.
- `work/inbox/{evidence,concept,merge,query_feedback}_suggestions.jsonl`
  — cross-talk channels.
- `work/index.md` — regenerated by `work tend`.
- `concept_created`, `evidence_added`, `inbox_suggestion_created`,
  `inbox_consolidated`, `stage_changed` events.

## When to use

- Create a concept folder after extracting a seed.
- Take a claim before mutating per-concept files (writer fanout).
- Append evidence pulled from `corpus find`.
- Push a feedback record from a query workflow.
- Run `work tend` to consolidate the inbox, expire stale claims, and
  regenerate `work/index.md`.

## Exit codes

`work claim` and `work release` exit 2 on contention. `work add`
verbs exit 1 on validation failure (bad slug, malformed records,
missing file).
