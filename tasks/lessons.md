- **CLI file exploration design**: Avoid vague "CLI or shell" guidance for skill workflows -> make one path canonical. For wikify skills, wrap `ls`/`rg`/`cat` affordances inside the CLI so bundle reads are constrained, rendered, and logged; keep raw shell tools as debugging escape hatches.
- **Schema field renames sweep YAML and prompts**: Renaming `WriteRequest.evidence_v2 -> evidence` updated Python but missed `prompts/write.yaml`'s `input_schema` and `{evidence_v2}` placeholder. Grep `*.yaml` and `*.md` after every Pydantic field rename.
- **Bundle.open is strict; bootstrap constructs directly**: `Bundle.open(path)` requires `run/state.json`. Tests and helpers that just need a path-context dataclass construct `Bundle(root=...)` directly. The privileged bootstrap path is `run init`, which writes state.json then proceeds.
- **Metric without producer is a no-op**: Adding `M5_hit_rate` to eval is meaningless until `chunk_read` events are emitted. Ship the producer (`corpus find` / `corpus show`) in the same change as the consumer; wire an end-to-end test that proves overlap > 0.
- **Boundary loggers do not mutate the target pre-init**: For `run init` IO capture, tee to `tempfile.mkdtemp()` and promote files to `<bundle>/run/io/` only after init succeeds. A half-created `<b>/run/` is otherwise indistinguishable from a real bundle to layout detection.
- **No path-dependent qualifiers in active surface**: "v2 bundle", "post-pivot X", "atomic skill", "legacy fallback" describe history, not behavior. Live code/docs/skills should describe what something IS. Reserve historical framing for explicitly labelled history docs such as `tasks/skill-centric-redesign-plan.md`.
- **Drop dead duplication on rename**: When renaming `WriteEvidenceRefV2 -> WriteEvidenceRef`, the old `WriteEvidenceRef` (small form) was kept alive though no consumer read it. Rename + grep-for-callers + delete-orphan in the same change.
- **Don't ship dead commands in canonical docs**: `docs/filesystem-state-design.md` listed `wikify run set --corpus` and `corpus find --out/--in/...` flags that the actual CLI rejects. Doc examples must be derived from `--help` output, not aspiration.
- **Skill primitives must not smuggle workflows**: Bundle/setup/helper skills should expose unit operations and contracts, not bundled strategy choices like exploration order, readiness thresholds, or writer fanout. Workflow skills own those decisions.
- **MCP should not blindly mirror CLI verbs**: For Wikify, expose expressive corpus/wiki primitives plus resources, keep workflows in skills, and keep CLI as a first-class sibling adapter rather than describing it as legacy.
- **No meta-references in shipped code or docstrings**: Phrases like "per `tasks/mcp_plan.md`", "Phase 1 ships X", "in later phases", "see the plan" rot the moment the plan moves and contaminate the public surface with session-local framing. Source comments, docstrings, and skill docs must describe what the code IS and DOES. Historical motivation belongs in commit messages and explicitly labelled history docs only. Internal algorithm phases ("phase 1: cache lookup", "phase 2: bulk resolution") that describe code stages are fine.
- **PDF text-backend choice drives ligature recovery**: Docling's `DoclingParseDocumentBackend` reads PDF text without ToUnicode CMap awareness; `PyPdfiumDocumentBackend` does. For scientific papers (ligatures, special glyphs), prefer PyPdfium. Symptom: broken `fi`/`fl` in body text after default Docling parse.
- **SQLite `INSERT OR IGNORE` does not honor FK constraints**: It only suppresses unique-key violations. FK violations still raise `IntegrityError`. Filter mappings against valid parent rows BEFORE insert, not after. Surfaced as wave-G refresh FK error on stale `chunk.equation_ids`.
- **Mirror implementations of normalizers are a latent dedup bug**: `author_key` was reimplemented in 4 sites and drifted (one stripped hyphens, others didn't), so the same person hashed to different keys. Normalizers belong in ONE canonical module; everything else imports.
- **Parser-side filters: preserve over clean**: When in doubt, keep the chunk/author/figure and let downstream consumers filter via `exclude_kinds`, boilerplate flags, or caption presence. Aggressive boilerplate sets eat content (Acknowledgments, Author contributions, Data availability); aggressive digit-cluster strips eat 2-3 row table columns. Same idea for the formula leak gate: inspect what gets persisted (post-unescape), not the raw export.
- **Figure dedup by caption + aspect ratio, not raw count**: Total-figure count is a noisy quality signal — uncaptioned sub-panel duplicates inflate it. Captioned-figure count + captioned-ratio is the real measure. 44% -> 72% captioned ratio while losing 1318 uncaptioned thumbnails is a quality WIN.
- **Subagent self-reported tokens are unreliable**: A haiku/sonnet child cannot accurately introspect its tool-result intake; self-reported `tokens_in` (e.g. 1.2k–7k) routinely undershoots the harness `<usage><total_tokens>` (35–50k) by 5–10×. Use the harness usage totals captured at the Agent tool boundary for telemetry; do not trust subagent self-reports.
- **`context_show().health` lacks rank_metrics**: The wikify-baseline SKILL.md claims `context_show().health.rank_metrics.{source,author}` exists so persons can be seeded without an extra call. The current MCP response only exposes `field_scores`. Either expose rank_metrics or drop the SKILL.md promise; until then, plan on a separate `corpus_find --by author` call for person seeding.
- **`wikify work show` evidence count is stale**: After `wikify work build-evidence` writes to `evidence.jsonl`, `wikify work show <slug>` keeps reporting `evidence_chunks: 0` until something invalidates the work-card cache. Always re-confirm by `wc -l evidence.jsonl` rather than trusting `work show`.
- **Vetter quote check fails on Unicode confusables**: Verbatim-substring check rejects chunks whose body uses superscript numerals or special glyphs when the vetter quotes the rendered form. Normalize both sides (NFKC) before the literal compare; otherwise budget for ~1 record loss per slug containing chemistry/math notation.
- **Writer must not emit `\uXXXX` escapes**: When the writer emits literal six-character `–` strings instead of U+2013 en-dashes, the rendered markdown shows the escape, not the dash. JSON encoders already write UTF-8 — emit unicode characters directly. Add a `wikify draft check` lint that rejects literal `\\u[0-9a-f]{4}` patterns inside prose fields.
- **Eval is silent on missing telemetry**: `wikify eval` accepts an empty call timeline and reports `telemetry: 0 calls, 0.0 haiku-eq` without complaining. The baseline SKILL.md asks for `record-calls` per wave but the gate is advisory only — `wikify run close` should warn (or fail) when no `call` events exist on the timeline.
